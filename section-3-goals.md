# Section 3 — Goals (1,200 chars per goal)

## Source Material

Load current goals from the path specified in `goals_path` in USER-CONFIG. Cross-reference with the goals established in your prior Connect review (loaded from `past_review_path`).

## Structure

For each goal:
- **Name the goal** with a percentage weighting to signal priority (e.g., "Goal #2. Drive Governance (50%)")
- Use 3-4 goals total, each treated as a standalone unit
- Within each goal, use a consistent internal structure:
  - **What:** What you plan to deliver. One sentence.
  - **How:** How you will execute and with whom. One sentence.
  - **Measures:** What success looks like, expressed in measurable terms. 3-5 bullets.

## Format

```
Goal #[N]. [Title] ([X]%)
What: [What you plan to deliver]
How: [How you will execute and with whom]
Measures:
  - [Metric with target]
  - [Metric with target]
  - [Metric with target]
[One sentence on dependencies or coordination required]
```

## Rules

- **Compliance goal** (Goal #1) is auto-assigned language; use the standard Microsoft compliance goal text
- **At least one goal must reference security** — governance compliance and ADO controls both qualify
- Ensure goals clearly build forward from past work, rather than introducing unrelated new areas
- Show focus, not a laundry list — when you say yes to something, what are you saying no to
- Character limit is 1,200 per goal (applied individually, not across all goals)
